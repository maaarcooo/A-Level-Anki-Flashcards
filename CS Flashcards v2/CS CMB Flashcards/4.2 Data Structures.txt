What is an array? | An ordered, finite set of elements of a single type
What type of elements can an array contain? | Elements of a single type only
What is a 1D (one-dimensional) array? | A linear array
What is zero-indexed? | When the first element in an array is at position zero
Unless stated otherwise, how are arrays indexed? | Zero-indexed (first element at position 0)
How can a two-dimensional array be visualised? | As a table or spreadsheet
When searching a 2D array, what order do you follow? | First go down the rows, then across the columns
How can a three-dimensional array be visualised? | As a multi-page spreadsheet or multiple 2D arrays
What syntax is used to select an element in a 3D array? | threeDimensionalArray[z,y,x] where z is array number, y is row, x is column
What is a record? | A row in a file made up of fields
Where are records commonly used? | In databases
What is a field? | A component/attribute within a record
How is a field in a record identified? | Using recordName.fieldName syntax
What is a list? | A data structure consisting of a number of ordered items where items can occur more than once
How are list values stored compared to arrays? | Non-contiguously (not next to each other in memory)
What is non-contiguous storage? | When data items do not have to be stored next to each other in memory
Can lists contain elements of more than one data type? | Yes, unlike arrays
What does isEmpty() do on a list? | Checks if the list is empty
What does append(value) do on a list? | Adds a new value to the end of the list
What does remove(value) do on a list? | Removes the value the first time it appears in the list
What does search(value) do on a list? | Searches for a value in the list and returns a Boolean
What does length() do on a list? | Returns the length of the list
What does index(value) do on a list? | Returns the position of the item in the list
What does insert(position, value) do on a list? | Inserts a value at a given position
What does pop() do on a list? | Returns and removes the last value in the list
What does pop(position) do on a list? | Returns and removes the value at the given position
What is a tuple? | An ordered set of values of any type that is immutable
What does immutable mean? | Cannot be changed - elements cannot be added or removed once created
How are tuples initialised? | Using regular brackets () instead of square brackets []
What happens if you try to change a value in a tuple? | A syntax error occurs
What is a linked list? | A dynamic data structure used to hold an ordered sequence where items don't need to be in contiguous locations
What is a dynamic data structure? | A data structure that can grow or shrink in size during program execution
What is a node in a linked list? | Each item in a linked list containing a data field and a pointer field
What does the data field in a linked list contain? | The value of the actual data which is part of the list
What does the pointer field in a linked list contain? | The address of the next item in the list
What two pointers does a linked list store? | The index of the first item (Start) and the index of the next available space (NextFree)
How does traversal of a linked list work? | Begins at the Start pointer index and outputs values at each node until the pointer field is empty/null
What indicates the end of a linked list has been reached? | When the pointer field is empty or null
How are values added or removed in a linked list? | By editing pointers
What is a disadvantage of removing nodes from a linked list? | The node is only ignored, not truly removed, which wastes memory
Why do linked lists require more memory than arrays? | Because storing pointers requires additional memory
Can items in a linked list be directly accessed? | No, they can only be traversed in sequence
What is random access? | The ability to access a specific element directly given its index (possible in arrays, not linked lists)
What is a graph? | A set of vertices/nodes connected by edges/arcs
What are vertices in a graph? | The nodes or points in a graph
What are edges in a graph? | The connections between nodes (also called arcs)
What is a directed graph? | A graph where edges can only be traversed in one direction
What is an undirected graph? | A graph where edges can be traversed in both directions
What is a weighted graph? | A graph where a cost is attached to each edge
What two methods can computers use to process graphs? | Adjacency matrix and adjacency list
What is an adjacency matrix? | A 2D array representation of a graph showing connections between nodes
What is an adjacency list? | A list representation showing each node and its connected nodes with weights
What are the advantages of an adjacency matrix? | Convenient to work with, quicker access times, easy to add nodes
What is the advantage of an adjacency list? | More space efficient for large, sparse networks
What is a stack? | A last in first out (LIFO) data structure
What does LIFO stand for? | Last In First Out
Where can items be added or removed in a stack? | Only from the top of the stack
Give two uses of stacks | Reversing actions (e.g., back button in browsers) and undo functionality in applications
Can a stack be static or dynamic? | Either - it can be implemented as a static or dynamic structure
When are static stacks preferred? | When the maximum size required is known in advance
Why are static stacks preferred when size is known? | They are easier to implement and make more efficient use of memory
What pointer do stacks use? | A pointer pointing to the top of the stack where the next data will be inserted
What does isEmpty() do on a stack? | Checks if the stack is empty by checking the value of the top pointer
What does push(value) do on a stack? | Adds a new value to the top of the stack (after checking stack is not full)
What does peek() do on a stack? | Returns the top value from the stack without removing it
What does pop() do on a stack? | Removes and returns the top value of the stack
What does size() do on a stack? | Returns the size of the stack
What does isFull() do on a stack? | Checks if the stack is full by comparing stack size to the top pointer
What is a queue? | A first in first out (FIFO) data structure
What does FIFO stand for? | First In First Out
Where are items added in a queue? | At the end (back) of the queue
Where are items removed from in a queue? | From the front of the queue
Give three common uses of queues | Printers, keyboards, and simulators
What two pointers do queues use? | One pointing to the front and one pointing to the back of the queue
What is a linear queue? | A queue implemented using an array where items are added to the next available space starting from the front
Why is a linear queue ineffective? | Because addresses used by removed items cannot be reused
What is a circular queue? | A queue where the rear pointer can loop back to the front of the array when it reaches the maximum size
How does a circular queue differ from a linear queue? | The rear pointer can wrap around to reuse empty spaces at the front
What is the advantage of a circular queue over a linear queue? | More effective use of space in the array
What is the disadvantage of a circular queue? | Harder to implement than a linear queue
What does enQueue(value) do? | Adds a new item to the end of the queue and increments the back pointer
What does deQueue() do? | Removes the item from the front of the queue and increments the front pointer
How does isEmpty() work for a queue? | Compares the front and back pointers
How does isFull() work for a queue? | Compares the back pointer and queue size
What is a tree? | A connected form of a graph with a root node and nodes connected by branches
What is the root node? | The top node in a tree with no incoming edges
What is an edge in a tree? | A connection between two nodes (also called branch or arc)
What is a child node? | A node with incoming edges
What is a parent node? | A node with outgoing edges
What is a subtree? | A subsection of a tree consisting of a parent and all its children
What is a leaf node? | A node with no children
What is a binary tree? | A tree where each node has a maximum of two children
Why are binary trees used for binary searches? | Because information in these trees is easy to search through
How is a binary tree commonly represented? | By storing each node with a left pointer and a right pointer
What data structure is typically used to implement a binary tree? | Two-dimensional arrays
What are the three methods of traversing a binary tree? | Pre-order, In-order, and Post-order
What is the outline method? | A technique for remembering tree traversal orders by tracing around the tree
What order does pre-order traversal follow? | Root node, left subtree, right subtree
In the outline method, how is pre-order traversal performed? | Nodes are traversed in the order you pass them on the left
What is pre-order traversal useful for? | Programming languages where the operation is written before the values (prefix notation)
What order does in-order traversal follow? | Left subtree, root node, right subtree
In the outline method, how is in-order traversal performed? | Nodes are traversed in the order you pass under them
What is in-order traversal useful for? | Traversing nodes in sequential order
What order does post-order traversal follow? | Left subtree, right subtree, root node
In the outline method, how is post-order traversal performed? | Nodes are traversed in the order you pass them on the right
What is a hash table? | An array coupled with a hash function
What does a hash function do? | Takes in data (a key) and outputs a hash value that maps to an index
What is the key in hashing? | The input data given to the hash function
What is the hash in hashing? | The output value from the hash function
What is the role of the hash function in a hash table? | To map the key to an index in the hash table
What is a collision in a hash table? | When two different inputs result in the same hashed value
What should a good hashing algorithm have? | A low probability of collisions occurring
What is collision resolution? | Placing an item in the next available location when a collision occurs
Why are hash tables commonly used for indexing? | They provide fast access to data due to keys having a unique, one-to-one relationship with storage addresses
What type of data structure is an array? | Static (fixed size)
What type of data structure is a linked list? | Dynamic (can grow/shrink)
What is the key difference between arrays and lists regarding data types? | Arrays contain single type only; lists can contain multiple data types
What is the key difference between arrays and lists regarding memory storage? | Arrays store data contiguously; lists store data non-contiguously
How do you access elements in an array? | Using an index in square brackets, e.g., array[3]
How do you access fields in a record? | Using dot notation: recordName.fieldName
What happens when you traverse to a null pointer in a linked list? | The traversal ends (end of list reached)
In a linked list, what must be updated when adding a new node? | The NextFree pointer and the pointer of the preceding node
In a linked list, what must be updated when removing a node? | The pointer of the preceding node (to bypass the removed node)
What check must be performed before push() on a stack? | Check that the stack is not full
What check must be performed before pop() on a stack? | Check that the stack is not empty
What check must be performed before peek() on a stack? | Check that the stack is not empty
How does the rear pointer behave in a circular queue when it reaches maxSize? | It loops back to position 0 (if that position is empty)
What is the formula for moving the rear pointer in a circular queue? | rearPointer = (rearPointer + 1) MOD maxSize
