What is the A* Algorithm? | A generalisation of Dijkstra's algorithm that approximately finds the shortest path between two nodes using a cost function (distance from source to current node) and a heuristic function (approximate distance from next node to destination).
What two functions does the A* algorithm use? | Cost function (distance travelled from source to current node) and heuristic function (approximate distance from next node to destination).
What is Big O Notation? | A way of classifying algorithms based on their complexity by setting an upper bound for the worst case space or time complexity as a mathematical function of input size.
What does Big O Notation measure? | The worst case space or time complexity of an algorithm as a function of input size.
What is Binary Search? | An O(log n) algorithm to search a sorted list for a particular item by repeatedly halving the sublist which could contain the item.
What is the time complexity of Binary Search? | O(log n)
What is a requirement for using Binary Search? | The list must be sorted.
What is Breadth-First Traversal? | A method of traversing a graph by using a queue to visit all neighbours of the current node before doing the same to each of the neighbours until the entire graph has been explored.
What data structure does Breadth-First Traversal use? | A queue.
What is Bubble Sort? | An O(n²) sorting algorithm that iterates through a list, comparing each element to its successor and swapping elements if the successor is greater than the current element, repeated until no more swaps can be made.
What is the time complexity of Bubble Sort? | O(n²)
What is Depth-First Traversal? | A method of traversing a graph by using a stack to travel as far along one route as possible, then backtracking and doing the same for the remaining routes until the entire graph has been explored.
What data structure does Depth-First Traversal use? | A stack.
What is Dijkstra's Shortest Path Algorithm? | An algorithm to find the shortest path between two nodes on a graph by using a priority queue to keep track of the shortest distance (cost) to each node from the starting node until the destination node is found.
What data structure does Dijkstra's algorithm use? | A priority queue.
What is Insertion Sort? | An O(n²) sorting algorithm that divides a list into a sorted and unsorted section, where elements from the unsorted section are compared with and placed in the correct position in the sorted section one by one.
What is the time complexity of Insertion Sort? | O(n²)
What is Linear Search? | An O(n) algorithm to search a list for a particular item by iterating through the list and checking each element until the required item is located or the end of the list is reached.
What is the time complexity of Linear Search? | O(n)
What is Merge Sort? | An O(n log n) divide-and-conquer sorting algorithm that recursively halves the list into sublists until all sublists are of length 1, then merges them back together in sorted order.
What is the time complexity of Merge Sort? | O(n log n)
What type of algorithm is Merge Sort? | A divide-and-conquer algorithm.
What is O(1) (Constant time/space)? | An algorithm that always takes a constant amount of time or memory space to execute, regardless of the input size.
What is O(2ⁿ) (Exponential time/space)? | An algorithm whose execution time or required memory space grows exponentially with input size (higher complexity than polynomial).
What is O(log n) (Logarithmic time/space)? | An algorithm whose execution time or required memory space grows logarithmically with input size (lower complexity than polynomial).
What is O(n) (Linear time/space)? | An algorithm whose execution time or required memory space is directly proportional to the size of its input.
What is O(nᵏ) (Polynomial time/space)? | An algorithm whose execution time or required memory space is proportional to the input size raised to the power of a constant (k), e.g. O(n²), O(n³).
What is Quick Sort? | An O(n log n) sorting algorithm that relies on choosing a pivot element when splitting the list; elements greater than the pivot go into one sublist, the rest go into the other.
What is the time complexity of Quick Sort? | O(n log n)
What is a pivot in Quick Sort? | An element chosen when splitting the list; elements greater than the pivot go into one sublist, while the rest go into the other.
What is Space Complexity? | A measure of the amount of memory space needed by an algorithm to solve a particular problem of a given input size.
What is Time Complexity? | A measure of the amount of time needed by an algorithm to solve a particular problem of a given input size.
An algorithm that always takes a constant amount of time regardless of input size has what complexity? | O(1) - Constant time.
An algorithm whose execution time grows exponentially with input size has what complexity? | O(2ⁿ) - Exponential time.
An algorithm whose execution time grows logarithmically with input size has what complexity? | O(log n) - Logarithmic time.
An algorithm whose execution time is directly proportional to input size has what complexity? | O(n) - Linear time.
An algorithm whose execution time is proportional to input size raised to a constant power has what complexity? | O(nᵏ) - Polynomial time.
Which searching algorithm requires a sorted list? | Binary Search.
Which searching algorithm works on unsorted lists? | Linear Search.
Which graph traversal uses a queue? | Breadth-First Traversal.
Which graph traversal uses a stack? | Depth-First Traversal.
Which sorting algorithms have O(n²) time complexity? | Bubble Sort and Insertion Sort.
Which sorting algorithms have O(n log n) time complexity? | Merge Sort and Quick Sort.
How does A* algorithm differ from Dijkstra's algorithm? | A* uses an additional heuristic function to estimate distance to the destination, making it faster by prioritising promising paths.
What happens when Bubble Sort completes? | No more swaps can be made, indicating the list is fully sorted.
In Insertion Sort, how is the list divided? | Into a sorted section and an unsorted section.
When does Binary Search terminate? | When the item is found or the sublist cannot be halved further (item not present).
What makes Merge Sort a divide-and-conquer algorithm? | It recursively divides the list into sublists of length 1, then conquers by merging them back in sorted order.
How does Quick Sort achieve sorted sublists of length 1? | By repeatedly partitioning around pivots until all sublists contain single elements, which are inherently sorted.
