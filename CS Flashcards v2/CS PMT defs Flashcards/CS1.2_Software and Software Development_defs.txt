What is BIOS (Basic Input Output System)? | A program that initialises and tests system hardware functionality, then loads the OS from hard disk into RAM when the computer is turned on
What program initialises hardware and loads the OS into RAM at startup? | BIOS (Basic Input Output System)
What are device drivers? | Programs supplied with peripheral devices that allow the OS to control and communicate with the device
What allows the OS to control and communicate with peripheral devices? | Device drivers
What is a distributed operating system? | An OS spread over multiple computer servers on a network, acting as a single system to parallel process a job
What type of OS spreads across multiple network servers to parallel process jobs? | Distributed operating system
What is an embedded operating system? | A specialised OS with limited resources and functionality, built in to control a single machine
What type of OS is built into a single machine with limited functionality? | Embedded operating system
What is First Come First Served scheduling? | A scheduling algorithm where processes are dealt with in the order they arrive (a queue)
Which scheduling algorithm processes jobs in arrival order like a queue? | First Come First Served
What is intermediate code? | Code partly translated between high-level and machine language, produced by a compiler
What produces intermediate code? | A compiler
What are interrupts? | Signals from hardware, software or the clock to alert the CPU
What happens if an interrupt has higher priority than the current task? | The current routine pauses and resumes after the interrupt is executed
What are Interrupt Service Routines (ISR)? | When a higher priority interrupt occurs, register contents are transferred to the system stack at the end of the current FDE cycle and the interrupt is handled
Where are register contents stored when handling a higher priority interrupt? | The system stack
What is memory management? | The efficient organisation and allocation of main memory to programs in use
What is the purpose of memory management? | To efficiently organise and allocate main memory to programs in use
What is multi-level feedback queues scheduling? | A scheduling algorithm using multiple queues with different priorities, where jobs can be moved between queues
Which scheduling algorithm uses multiple priority queues and allows jobs to move between them? | Multi-level feedback queues
What is a multi-tasking operating system? | An OS capable of running multiple tasks simultaneously
What type of OS can run multiple tasks simultaneously? | Multi-tasking operating system
What is a multi-user operating system? | An OS with one mainframe computer and multiple terminals allowing multiple users to access resources, each given a time slice of the processor
How do users share resources in a multi-user OS? | Each user is given a time slice of the mainframe processor
What is an operating system? | A set of programs managing computer operation, loaded into RAM when turned on, bridging the user to the hardware
What bridges the user to the hardware? | The operating system
What is paging? | Partitioning memory into fixed sized physical divisions called pages
What are the fixed sized physical memory divisions called in paging? | Pages
What is a real time operating system? | An OS where data is processed as it comes, with responses generated within a guaranteed timeframe
Which type of OS guarantees response times? | Real time operating system
What is Round-Robin scheduling? | A scheduling algorithm where each process gets an equal time slice on a first in first out basis; unfinished processes rejoin the queue end
What happens to an unfinished process in Round-Robin scheduling? | It joins the end of the queue
What is scheduling? | Allocating processor time to each application to ensure processor time is used efficiently when multitasking
What is the purpose of scheduling? | To ensure processor time is used efficiently when multitasking
What is segmentation? | Partitioning memory into variable sized logical divisions called segments
What is the difference between paging and segmentation? | Paging uses fixed sized physical divisions; segmentation uses variable sized logical divisions
What is Shortest Job First scheduling? | A scheduling algorithm that picks the process with shortest estimated running time and runs it until completion
Which scheduling algorithm runs the job with shortest estimated time until completion? | Shortest Job First
What is Shortest Remaining Time scheduling? | A scheduling algorithm that picks the process with shortest estimated time remaining; switches if a shorter process is added
How does Shortest Remaining Time differ from Shortest Job First? | SRT can switch to a new process if it has shorter remaining time; SJF runs until completion
What is a virtual machine? | Any instance where software takes on the function of a machine, including intermediate code or running an OS within another
Give two examples of virtual machines | Running intermediate code; running an OS within another OS
What is virtual memory? | An allocated area of secondary storage where pages of inactive jobs are swapped to free up RAM for the current job
What is the purpose of virtual memory? | To free up RAM for the current job by swapping inactive pages to secondary storage
What is an application? | A program that can be run on a computer, allowing the user to carry out specific tasks
What is an assembler? | A translator that converts assembly language into machine code
What does an assembler translate? | Assembly language into machine code
What is closed source software? | Proprietary software sold with a licence and usage restrictions; source code not available to users
What distinguishes closed source from open source software? | Closed source has restricted licences and hidden source code; open source is freely available
What is code generation? | The third and final stage of compilation where equivalent machine code is produced
Which stage of compilation produces machine code? | Code generation (third stage)
What is compilation? | The process of analysing high level language source code and converting it into machine code
What is a compiler? | A translator that converts high level language to machine code
What is an interpreter? | A translator that checks source code for syntax errors line by line, translates to machine code and executes each line
How does an interpreter differ from a compiler? | An interpreter translates and executes line by line; a compiler translates the entire program first
What is lexical analysis? | The first stage of compilation where spaces and comments are removed, simple errors found, keywords/constants/variables replaced by tokens, and variable names loaded into the symbol table
What happens during lexical analysis? | Spaces and comments removed, simple errors found, tokens replace keywords/constants/variables, symbol table populated
What are libraries (in programming)? | Collections of pre-compiled programs that can be loaded into a program and run when required
What is a linker? | A program that places appropriate machine addresses in call and return instructions so all required object code files and modules are linked together
What is the purpose of a linker? | To link all required object code files and modules together with correct machine addresses
What is a loader? | A program that loads the executable object program and its associated libraries into memory before execution
What does a loader do? | Loads executable program and libraries into memory before running
What is open source software? | Software whose source code is freely available to view, redistribute or modify
What is optimisation (in compilation)? | During code generation, making object code as efficient as possible by removing redundancies while producing the same result
What is syntax analysis? | The second stage of compilation where statements, expressions and tokens are checked for syntax errors using syntax diagrams
Which compilation stage uses syntax diagrams to check for errors? | Syntax analysis (second stage)
What is a translator? | A program that converts code from one computer language to another
Name three types of translators | Assembler, compiler, interpreter
What are utilities? | System software with specific purposes related to maintenance such as optimising performance, diagnosing issues, backing up files, and setting up firewalls
What are Agile methodologies? | An iterative process producing incremental prototypes over short sprints, with user testing and feedback incorporated into future sprints
What characterises Agile development? | Short sprints, incremental prototypes, user testing, responsive to feedback
What is Extreme Programming? | An Agile methodology more responsive to changing user needs with short development cycles, frequent releases, and checkpoints for changing requirements
How does Extreme Programming differ from standard Agile? | More frequent releases, shorter cycles, checkpoints for requirement changes
What is Rapid Application Development (RAD)? | A method with speedy responses to changes through repeated prototyping, continual evaluations, and strict time limits; users quickly receive reduced-functionality mockups
What does the user receive quickly in RAD? | A reduced-functionality mock up of the program
What is the Spiral Model? | An iterative version of Waterfall where stages are refined and repeated; first cycle produces initial prototype, successive cycles produce refined prototypes
How does Spiral Model differ from Waterfall? | Spiral iterates and refines stages; Waterfall is linear
What is the Waterfall Lifecycle? | Development stages completed one at a time in linear order; results from each stage input to the next; can return to previous stages but must rework following stages
What is a disadvantage of returning to a previous stage in Waterfall? | All following stages must be worked through again
What is assembly language? | A low level language closely related to machine code, using descriptive names and mnemonics for instructions, related to computer design
What makes assembly language more advanced than machine code? | It uses descriptive names and mnemonics for instructions
What are attributes (in OOP)? | Data recorded as a variable associated with an object
What is a class (in OOP)? | A template defining the attributes and methods used to create objects
What defines a class? | Attributes and methods
What is direct addressing? | The simplest addressing mode where the operand stores the memory address of the value to be operated on
In direct addressing, what does the operand contain? | The memory address of the value to be operated on
What is encapsulation? | A method of maintaining data integrity by only allowing class methods to access data in an object's attributes
What is the purpose of encapsulation? | To maintain data integrity
What is immediate addressing? | An addressing mode where the data in the address field is constant and the operand is the actual value to be operated on
In immediate addressing, what does the operand contain? | The actual value to be operated on (not an address)
What is indexed addressing? | An addressing mode where the operand address is calculated by adding a constant from the index register to the absolute address in the instruction
How is the address calculated in indexed addressing? | Index register constant + absolute address in instruction
What is indirect addressing? | An addressing mode where the operand stores the address of the location holding the memory address of the required data
How many levels of addressing does indirect addressing use? | Two levels (operand points to address which points to data)
What is inheritance (in OOP)? | The concept of subclasses inheriting methods and attributes from their parent class (super class)
What does a subclass inherit from its parent class? | Methods and attributes
What are methods (in OOP)? | Program subroutines representing actions an object can perform
What is the difference between methods and attributes? | Methods are actions/subroutines; attributes are data/variables
What are object-oriented languages? | Programming languages where code is made of objects (class instances) with their own attributes and behaviours that can interact
What is an object (in OOP)? | An instance of a class whose behaviour depends on how its attributes were defined
What is polymorphism? | The ability for objects of different classes to use the same method to perform an action
What does polymorphism allow? | Objects of different classes to use the same method name
What are procedural languages? | High level languages where statements are grouped into functions and procedures, with built in data types and data structures
What are statements grouped into in procedural languages? | Functions and procedures
What are programming paradigms? | Styles of computation and programming, chosen according to the problem at hand
What determines which programming paradigm to use? | The problem at hand
